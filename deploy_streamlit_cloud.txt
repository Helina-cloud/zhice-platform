智策 · 部署到 Streamlit Community Cloud（简要步骤）

1) 把代码推到 GitHub（公开仓库免费档可用；私有需付费档）。

2) 打开 https://share.streamlit.io ，用 GitHub 登录，「New app」。

3) 填写：
   - Repository：你的仓库
   - Branch：main（或你的默认分支）
   - Main file path：streamlit_app.py

4) Advanced settings → Python version：与仓库根目录 runtime.txt 一致（当前为 3.11）。

5) App secrets（必须是合法 TOML，否则会提示 Invalid format）

   可直接粘贴下面模板（英文双引号；冒号后空格；每行一条；不要中文标点引号）：

   OPENAI_API_KEY = "paste-your-key-here"
   OPENAI_API_BASE = "https://api.deepseek.com"
   CHAT_MODEL = "deepseek-chat"
   EMBEDDING_PROVIDER = "huggingface"

   注意：
   - 字符串必须用英文双引号 "..."，不要用中文引号 「」 "" ''
   - 不要用 export KEY=... 这种 shell 写法
   - 密钥里若有双引号，整段密钥改用单引号包裹：OPENAI_API_KEY = 'sk-..."...'
   - 不要在句尾加逗号；不要用 # 注释写在半截引号里
   - 不要写 OPENAI_API_KEY = ""（空字符串仍会生效）；不用对话密钥时请删掉该行，
     勿留占位空串，否则会干扰嵌入时的 Authorization（云端嵌入易报 401）。

   嵌入（二选一）：
   - 走云端 OpenAI 兼容嵌入：顶层键名必须为 `EMBEDDING_API_KEY`、`EMBEDDING_API_BASE`
     （例如 Base = "https://api.openai.com/v1"）。若写在 TOML 节里，须能展开成上述环境变量名
     （如 [embedding] 下 api_key → EMBEDDING_API_KEY）。
   - 走本地向量：设 `EMBEDDING_PROVIDER = "huggingface"`；仅 DeepSeek 聊天且未单独配嵌入密钥时，
     程序也会自动改用 HuggingFace。

   保存后重新 Deploy。应用启动时会：
   - 把这些变量注入环境变量（inject_streamlit_secrets）
   - 若 data/vector_db 不存在，自动跑一次 ingest

6) 刷新页面重试。若出现 503：多为上游 LLM/中转繁忙，与向量库无关，稍后重试或换模型。

7) 本地调试仍用：streamlit run streamlit_app.py

8) 若本地 `python ingest.py` 正常，但云端嵌入报 401：
   - 逐项对照「本地 .env」与「App Secrets」：键名须一致（如 EMBEDDING_API_KEY），值须与本地相同；
     改 Secrets 后须 **Redeploy**，不要只刷新页面。
   - 仓库根目录 `runtime.txt` 与 Cloud Advanced → Python version 一致（避免依赖版本漂移）。
   - 新版已在注入 Secrets 时去掉首尾空格与 UTF-8 BOM；仍失败时在云雾控制台核对密钥与余额。
   - 可在 Secrets 增加一行：`OPENAI_HTTP_TRUST_ENV = false`（布尔型，见 TOML），避免托管环境代理变量干扰；
     部署后看页面错误里的诊断行 `OPENAI_HTTP_TRUST_ENV=...` 是否为 False。
   - 若仍 401：部分中转商限制数据中心出口 IP，家用网络能通过、云端不能；可改用 `EMBEDDING_PROVIDER=huggingface`
     或向中转商确认云端是否可用。
