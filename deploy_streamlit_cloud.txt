智策 · 部署到 Streamlit Community Cloud（简要步骤）

1) 把代码推到 GitHub（公开仓库免费档可用；私有需付费档）。

2) 打开 https://share.streamlit.io ，用 GitHub 登录，「New app」。

3) 填写：
   - Repository：你的仓库
   - Branch：main（或你的默认分支）
   - Main file path：streamlit_app.py

4) Advanced settings → Python version：与仓库根目录 runtime.txt 一致（当前为 3.11）。

5) App secrets（必须是合法 TOML，否则会提示 Invalid format）

   推荐：对话与嵌入均用 OpenAI，同一密钥（不必再配 EMBEDDING_API_*）：

   OPENAI_API_KEY = "sk-你的OpenAI密钥"
   OPENAI_API_BASE = "https://api.openai.com/v1"
   CHAT_MODEL = "gpt-4o-mini"
   EMBEDDING_PROVIDER = "openai"
   EMBEDDING_MODEL = "text-embedding-3-small"

   注意：
   - 字符串必须用英文双引号 "..."，不要用中文引号 「」 "" ''
   - 不要用 export KEY=... 这种 shell 写法
   - 密钥里若有双引号，整段密钥改用单引号包裹：OPENAI_API_KEY = 'sk-..."...'
   - 不要在句尾加逗号；不要用 # 注释写在半截引号里
   - 不要写 OPENAI_API_KEY = "" 占位；不用时请删掉该行

   嵌入（可选）：
   - 与上面共用 OpenAI：不设 `EMBEDDING_API_KEY` / `EMBEDDING_API_BASE` 即可。
   - 嵌入走另一兼容网关：再设顶层键 `EMBEDDING_API_KEY`、`EMBEDDING_API_BASE`。
   - 无 OpenAI 嵌入额度：设 `EMBEDDING_PROVIDER = "huggingface"`（需依赖 sentence-transformers）。

   保存后重新 Deploy。应用启动时会：
   - 把这些变量注入环境变量（inject_streamlit_secrets）
   - 若 data/vector_db 不存在，自动跑一次 ingest

6) 刷新页面重试。若出现 503：多为上游 LLM 繁忙，稍后重试或换模型。

7) 本地调试仍用：streamlit run streamlit_app.py

8) 若云端 ingest 报错（401 / 404 等）：
   - 对照仓库根目录 `.env.example` 与 App Secrets，改完后 **Redeploy**（不要只刷新页面）。
   - `runtime.txt` 与 Cloud Advanced → Python version 一致。
   - 可设 `OPENAI_HTTP_TRUST_ENV = false`（TOML 小写布尔）排除托管环境代理干扰；注入后为字符串，项目已做 false/true 归一化。
   - 仍失败时到 OpenAI 控制台核对密钥、计费与模型可用性。
